# Directives

SOPs (Standard Operating Procedures) written in Markdown.

Each directive defines:
- **Goal** — What are we trying to accomplish?
- **Inputs** — What data/context is needed?
- **Tools/Scripts** — Which `execution/` scripts to call?
- **Outputs** — What is the expected deliverable?
- **Edge Cases** — Known gotchas, API limits, error handling

## Conventions
- One directive per task type (e.g., `scrape_website.md`, `generate_report.md`)
- Update directives as you learn — they are living documents
- Never delete a directive without asking the user first
